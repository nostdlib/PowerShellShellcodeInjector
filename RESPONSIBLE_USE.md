# Responsible Use Policy

## Purpose

This project exists to support the following legitimate activities:

- **Education** -- learning how Windows memory management, shellcode loading, and position-independent code execution work at a low level.
- **Authorized security testing** -- penetration testing and red team engagements conducted with explicit written permission from the system owner.
- **Capture The Flag (CTF) competitions** -- solving challenges in controlled, sanctioned environments.
- **Security research** -- studying offensive techniques to develop better defenses, detection signatures, and security tooling.

## Prohibited Uses

The following uses of this project are strictly prohibited:

- **Unauthorized access** -- using this tool against any system, network, or application without explicit written authorization from its owner.
- **Malware development or distribution** -- incorporating this code into malicious software intended to harm, exploit, or compromise systems without authorization.
- **Data theft or exfiltration** -- using this tool to steal, intercept, or exfiltrate data from systems you do not own or have permission to test.
- **Denial of service** -- using this tool to disrupt, degrade, or deny service to any system or network.
- **Evasion of legal controls** -- using this tool to circumvent security controls in violation of applicable law or contractual obligations.
- **Distribution to malicious actors** -- knowingly providing this tool or derivatives to individuals or groups who intend to use it for unauthorized purposes.

## Reporting Vulnerabilities

If you discover a security vulnerability in this project itself (not in systems you are testing with it), please report it responsibly:

1. **Do not open a public issue.** Security vulnerabilities should not be disclosed publicly before a fix is available.
2. **Contact the maintainers privately** via the repository's security advisory feature or by email if one is listed.
3. **Provide sufficient detail** to reproduce the issue, including steps, environment, and potential impact.
4. **Allow reasonable time** for the maintainers to investigate and release a fix before any public disclosure.
